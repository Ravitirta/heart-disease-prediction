## Business Problem

Heart disease remains one of the leading causes of death worldwide, and early
screening is often limited by clinician time and the cost of invasive
diagnostic procedures. A well-calibrated predictive model, built on routinely
collected clinical measurements (blood pressure, cholesterol, exercise
stress-test results), can act as a **triage layer** — flagging patients who
warrant closer follow-up before symptoms become severe.

The critical constraint in this domain is **asymmetric error cost**: a
**False Negative** (telling a sick patient they are healthy) can delay
life-saving treatment, while a **False Positive** only costs an extra check-up.
A model graded purely on accuracy or F1-score can hide this asymmetry — so
this project treats the decision threshold itself as a tunable, business-aware
parameter rather than a fixed statistical default.

## What Makes This Project Different

| Generic ML Project | This Project |
|---|---|
| Tunes hyperparameters, reports one accuracy number | Tunes hyperparameters **and** the decision threshold, reporting both a leaderboard-optimal and a clinically-safe operating point |
| Engineers features by trial-and-error | Engineers features from documented cardiology domain knowledge (ischemia markers, metabolic syndrome clustering, age-normalized heart rate) |
| Treats missing/invalid values with a quick `.fillna()` | Explicitly flags clinically-impossible zero values as a signal, then imputes using **train-only statistics** to prevent data leakage |
| Outputs a raw probability | Outputs a probability **plus** a human-readable clinical rationale, generated by a locally-hosted LLM grounded in deterministic, auditable rule-based risk factors |

## Tech Stack

- **Data Processing:** Pandas, NumPy
- **Modeling:** Scikit-learn, XGBoost, LightGBM, CatBoost
- **Hyperparameter Optimization:** Optuna (TPE sampler)
- **Ensembling:** Soft Voting & Stacking (`StackingClassifier`)
- **Explainable AI:** Local LLM reasoning via Transformers (`Qwen2.5-0.5B-Instruct`)
- **Visualization:** Matplotlib, Seaborn
- **Environment:** Jupyter Notebook

## How to Run

```bash
# 1. Clone the repository
git clone https://github.com/<your-username>/heart-disease-prediction.git
cd heart-disease-prediction

# 2. Set up the environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 3. Place your dataset
#    data/raw/train.csv  and  data/raw/test.csv  (semicolon-separated)

# 4. Run the notebooks in order
jupyter notebook notebooks/01_Data_Cleaning_and_Imputation.ipynb
# → then 02, 03, 04, 05 in sequence
```

Each notebook reads its inputs from disk and writes its outputs back to disk,
so the pipeline is fully **reproducible and resumable** — you can re-run any
single stage without re-executing the entire pipeline.

### Clinical Safety: Dual-Threshold Decision Policy

Rather than reporting a single accuracy figure, the final ensemble is
calibrated at **two distinct operating points** from the same trained model:

| Strategy | Optimization Target | Trade-off |
|---|---|---|
| **Leaderboard** | Maximize F1-Macro | Best "headline" score; not tuned for missed-diagnosis risk |
| **Clinical Safety** | Recall (disease class) ≥ 95% | Deliberately lowers the decision threshold to minimize **False Negatives**, accepting more follow-up tests in exchange for far fewer missed diagnoses |

> **After running Notebook 04 on your dataset**, replace this line with
> your own measured result, e.g.: *"Switching from the Leaderboard threshold
> to the Clinical-Safety threshold reduced False Negatives from **X** to **Y**
> on the out-of-fold predictions — a **Z%** reduction in missed diagnoses,
> at the cost of [N] additional false positives."*

## Key Engineering Decisions

- **Anti-leakage imputation** — medians for `RestingBP` and `Cholesterol` are
  computed strictly from the training partition, then applied to test data,
  avoiding the common mistake of fitting statistics on the full dataset.
- **Domain-first feature engineering** — `Ischemia_Risk`, `Metabolic_Risk`,
  and `Angina_Oldpeak` encode cardiology reasoning, not arbitrary
  feature-crossing.
- **Threshold as a business decision, not a default** — the 0.5 cutoff is
  treated as one option among several, with an explicit clinically-motivated
  alternative.
- **Grounded explainability** — the LLM narrates pre-computed, auditable risk
  flags instead of free-form reasoning, bounding hallucination risk while
  still gaining natural-language fluency.
